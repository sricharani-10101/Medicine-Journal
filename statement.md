# Problem Statement
Minor health issues such as a common cold, headache, or acidity are common in everyday life that do not strictly require a hospital visit but do need immediate attention or over-the-counter medication. 
Searching on the internet for these simple remedies can often lead to overwhelming ,complex and unnecessary information. 
There is a need for a straightforward, accessible tool that provides quick, reliable  and easily available suggestions for medication and simple what to avoid for specific-common symptoms without confusion.

# Scope of Variable
1. Variable Name: symptom
2. Function: It stores the text the user types in.
3. Limited to: The code only works if symptom is exactly one of these words:
cold, headache, fever, body pain, cough, sore throat, stomach ache, cramps, acidity, vomiting, motion sickness, diarrhoea, allergy, cut, burn.
4. Do not work for: If symptom is anything else (like "broken leg" or "flu"), it suggests to visit a doctor.

# Targeted User
1. Anyone in need of common medical medication
2. Students
3. Simple hand medicine users

# High-Level Features
1. Symptom Based Diagnosis 
2. Comprehensive Advice System limited to Medication and avoidance list
3. Simple and handy medicines which are commonly used 
4. Command Interface
5. Common diseases based data
6. Alert to go to doctor if a symptom is out of data
7. Simple easy outreach of medication
